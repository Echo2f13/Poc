# Team Approach — NestJS Modular POC

We began by studying the NestJS framework as a team, focusing on its modular architecture, dependency injection, and decorator-based routing. After a brief knowledge-sharing session, we collectively decided on a feature-based folder structure with three core backend modules — Users, Products, and Orders — each owned by a dedicated developer. The backend team (Geethu, Chirag, Anirudh) built their modules in parallel once Manish Dev completed the project scaffold and root module wiring. On the frontend side, we split responsibilities by page: M Nikhil handled the login flow, Navya built the product catalog, and Rajshekar owned the orders page along with all shared CSS and utility functions. Satwik G independently developed the admin panel as a read-only dashboard that consumes all three REST endpoints. We used a clear integration order — backend modules first, then frontend pages — with daily syncs to resolve cross-module dependency issues like service exports and API contracts. The final integration and end-to-end testing was done collectively, with each member verifying their own component against the running server before merging into main.
